Authentication 4 features
#FeatureDescription
1 Login/login Unified login for all roles (Patient, HR, Admin). After authentication, middleware redirects based on role: HR/Admin → /hr/dashboard, Patient → /dashboard. Includes show/hide password, error messages, forgot password link, and loading state.
2 Register/register New patient account creation. Collects first/last name, email, mobile number (Philippine format), date of birth, password with confirmation. Full field validation with inline error messages.
3 OTP Verification/verify-otp 6-digit email OTP for registration, 2FA login, and password reset. Auto-verifies on complete entry. 60-second countdown before resend. Purpose-aware header text.
4 Forgot Password/forgot-password 3-step recovery: (1) Enter email → (2) Verify OTP → (3) Set new password with confirmation. Each step has back navigation and full error handling.

Access & Authentication 3 features
#FeatureDescription
39 Role-Based Login/login Shared login page for all roles. After authentication, middleware automatically redirects: HR role → /hr/dashboard, Admin role → /hr/dashboard, Patient role → /dashboard. No separate HR login URL needed.
40 HR Route Protection/hr/* All /hr/* routes require an active session with role HR or ADMIN. Unauthenticated users are redirected to /login. Patients who try to access /hr/* are redirected to /dashboard. Protected at middleware level — no client-side bypass possible.
41 HR Sign OutHR Header Sign out button in the HR portal user dropdown. Clears the session and redirects to the shared /login page.
