Authentication 4 features
# Feature Description
1 Login /login Patient signs in via email + password. Includes show/hide password toggle, descriptive error messages, forgot password link, and loading state on submit.
2 Register /register New patient account creation. Collects first/last name, email, mobile number (Philippine format), date of birth, password with confirmation. Full field validation with inline error messages.
3 OTP Verification /verify-otp 6-digit email OTP verification used for registration, login 2FA, and password reset. Auto-verifies on complete entry. Has 60-second countdown before resend is allowed. Shows purpose-aware header text.
4 Forgot Password /forgot-password 3-step password recovery: (1) Enter email → (2) Verify 6-digit OTP sent to email → (3) Set new password with confirmation. Each step has back navigation and full error handling.

Dashboard 6 features
# Feature Description
5 Welcome Banner /dashboard Personalized greeting that changes based on time of day (Good morning / Good afternoon / Good evening) with the patient's first name.
6 Active Visit Banner /dashboard Shows a live status banner when a patient has an ongoing visit. Displays current station, patients ahead, and estimated wait time. Only visible during an active visit.
7 Stats Summary Cards /dashboard 4 summary cards showing: Total Visits (all time), Lab Results count, Pending Payments count, and Unread Notifications count.
8 Recent Visits /dashboard Shows the last 3 visits with date, status badge, and a quick link to the full visit detail page.
9 Recent Results /dashboard Shows the last 3 lab/imaging results with result type, date, and release status badge.
10 Pending Payments /dashboard Highlighted section showing unpaid balances. Each item has the visit code, amount due, and a direct link to the payment detail page.

Visits 4 features
# Feature Description
11 Visit List /visits Paginated list of all patient visits. Shows 10 per page with previous/next navigation. Displays total count, visit date, queue code, status, and patient type.
12 Visit Search /visits Search bar to filter visits by queue code. Results update as the user types.
13 Visit Detail /visits/[code] Full visit summary showing: physician name, department, chief complaint, itemized services with costs, status badge, and payment status. Includes queue tracker if visit is active.
14 Visit Quick Links /visits/[code] From the visit detail page, one-click navigation to View Results, View Vitals, and View Payment for that specific visit.

Lab Results 4 features
# Feature Description
15 Results List /results Paginated list of all lab and imaging results with type filter pills: All, Laboratory, Imaging, Pathology. Shows 10 per page with total count.
16 Result Detail /results/[queueCode] Full result page showing test parameters table with: Test Name, Result Value, Reference Range, and Flag (H = High, L = Low, N = Normal) with color-coded indicators. Shows requesting doctor, release and verification timestamps.
17 PDF Download /results/[queueCode] Download or view the official lab result PDF file directly from the result detail page.
18 Result Status Badge /results/[queueCode] Displays the current online release status of the result (e.g. Released, Pending). Shows release and verification timestamps and the name of the verifying staff.

Vital Signs 2 features
# Feature Description
19 Vitals List /vitals Paginated list of all recorded vital sign entries. Displays date and a summary of key values per visit. Shows total record count with previous/next pagination.
20 Vitals Detail /vitals/[queueCode] Per-visit vitals breakdown showing: Blood Pressure (mmHg), Temperature (°C), Pulse Rate (bpm), Respiratory Rate (/min), Oxygen Saturation (%), Weight (kg), Height (cm), BMI, and Pain Scale (0–10). Each metric displays its normal reference range.

Payments 4 features
# Feature Description
21 Payment List /payments Paginated billing records with filter by status: All, Paid, Pending, Refunded. Displays 10 per page with total count, visit date, and amount.
22 Payment Detail /payments/[queueCode] Full billing breakdown with: itemized services table, discounts, HMO/PhilHealth coverage amounts, subtotal/tax/total calculation, payment method, cashier info, OR number, and a QR code for cashier scanning.
23 Payment Methods Display /payments/[queueCode] Shows the payment method used for each transaction: Cash, GCash, Maya, Credit Card, Bank Transfer, HMO, or PhilHealth.
24 Printable Receipt /payments/[queueCode]/receipt Print-friendly receipt page with hospital header, contact info, OR number, itemized services table, subtotal/discount/tax breakdown, total, cashier name, and signature line. Print button hidden on actual print output.

Notifications 5 features
# Feature Description
25 Notification Center /notifications Full notification list with filter tabs: All, Unread, Read. Each card is expandable to show full message content with timestamps.
26 Mark as Read / Unread /notifications Toggle read status on individual notifications. Unread notifications show a colored indicator dot. Read timestamp is recorded when marked as read.
27 Mark All as Read /notifications Single button to bulk-mark all notifications as read at once.
28 Notification Types /notifications Notifications are color-coded by type: Success (green), Warning (amber), Alert (red), Info (blue). Auto-detects context icons for Lab Results and Payment references.
29 Unread Badge Header / Sidebar Live unread count badge displayed on the notification bell icon in the header and on the Notifications item in the sidebar.

Real-Time & Navigation 6 features
# Feature Description
33 Queue Tracker Dashboard / Visit Detail Live polling widget showing the patient's current queue position, station name, number of patients ahead, and estimated wait time. Updates automatically without page reload.
34 Queue Alert Modal Global Full-screen pop-up alert triggered when the patient is called at a station. Shows queue number, station name, and call count. Plays an alert sound (muteable). Dismissed by clicking "Got it, I'm on my way!" or pressing Escape.
35 Collapsible Sidebar Global Desktop sidebar can be collapsed to an icon-only rail to maximize content viewing area. Tooltips appear on hover when collapsed. State is toggled via button in sidebar and header.
36 Responsive Mobile Sidebar Global On mobile and tablet, sidebar slides in as an overlay triggered by the hamburger menu button. Closes on overlay tap or navigation.
37 Dark / Light Mode Global Theme toggle in the header switches between dark and light mode. Preference persists across sessions.
38 Breadcrumb Navigation Global Header Header displays breadcrumb showing the current section (e.g. Patient Portal / Visits) for contextual navigation awareness.
